Estructura de Base de Datos (Supabase)

/* NÚCLEO MULTI-TENANT */
tenants          → cada negocio/servicio (pollería, consultorio, etc.)
tenant_configs   → configuración específica por tenant
plans            → Starter, Pro, Enterprise

/* USUARIOS */
users            → clientes finales
tenant_users     → relación usuario ↔ tenant

/* SERVICIOS IA */
conversations    → historial de chat por usuario + tenant
agent_tools      → herramientas disponibles por tipo de servicio

/* NEGOCIO */
orders           → pedidos (pollerías)
appointments     → citas (consultorios)
payments         → todos los pagos
menus            → carta por tenant (pollerías)

Flujo Completo: Chatbot Pollería con Agente IA

[1] CONVERSACIÓN LIBRE
    Cliente: "Quiero un combo familiar sin ensalada y 2 gaseosas XL"
    → Agente IA entiende, consulta menú del tenant
    → Agrega al carrito con modificaciones
    → Confirma: "Combo familiar sin ensalada (S/65) + 2 gaseosas XL (S/12) = S/77"

[2] PAGO
    Bot envía datos Yape/transferencia
    → Cliente envía captura
    → Sistema verifica → Pedido #0042 confirmado

[3] NOTIFICACIÓN COCINA (Telegram)
    n8n/API → grupo Telegram cocina:
    "🔔 PEDIDO #0042 — 7:45pm
     1x Combo Familiar sin ensalada
     2x Gaseosa XL
     📍 Delivery: Jr. Lima 234
     💰 Pagado S/77 ✅"

[4] SEGUIMIENTO
    Cocina escribe: "listo 0042"
    → Webhook detecta → API actualiza estado
    → WhatsApp cliente: "🛵 Tu pedido va en camino"

[5] ENTREGA + FEEDBACK
    Repartidor marca entregado
    → Cliente recibe confirmación
    → Bot pide reseña opcional ⭐
